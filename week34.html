Functions

Named Functions

A named function is a function declaration. A function declaration occurs when the function keyword is the first word of the statement. Functions assigned to a variable become function expressions.

Function Expression

A JavaScript function can also be defined using an expression. The function keyword can be used to define a function inside an expression. A function expression can be stored in a variable: var x = function (a, b) {return a * b}; After a function expression has been stored in a variable, the variable can be used as a function

Arrow Functions

An arrow function expression is a syntactically compact alternative to a regular function expression, although without its own bindings to the this, arguments, super, or new.target keywords. Arrow function expressions are ill suited as methods, and they cannot be used as constructors.

Other Topics

Optional Arguments

Many programming languages support optional function arguments - ie. functions in which certain arguments are optional. You can set a default value for any argument - so if the argument is not present, the default value will be used. Unfortunately, JavaScript is not one of these languages - so we will have to use a bit of trickery to get this effect in JavaScript. The simplest method of doing this is to check for the argument and create a new variable with the name of the argument if said argument is not present. For unlimited arguments, we provide an argument list when declaring a function. But we can avoid that and create a function that will take unlimited number of arguments. These arguments can be accessed using the 'arguments' variable.

Rest Parameters

The rest parameter has a prefix of three dots (...). A rest parameter allows us to represent an indefinite number of arguments as an array. The rest parameter can only be the last one. That’s the rule. It’s meant to capture the rest values.

Scope

In JavaScript there are two types of scope: Local and Global. Each function creates a new scope. Scope determines the accessibility (visibility) of these variables. Variables defined inside a function are not accessible (visible) from outside the function.
Local - variables declared within a JavaScript function
Global - variable declared outside a function

Nested Functions

Since all JavaScript functions are objects and they have a scope chain associated with them, they are, in fact, closures. Most commonly however, a closure is created when a nested function object is returned from the function within which it was defined. The nested function closures are a powerful feature of JavaScript and they are commonly used in advanced applications. For instance, it is commonly used when declaring event callbacks. Many JavaScript libraries make use of closures.
